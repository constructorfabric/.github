# Cyber Fabric

Cyber Fabric is an AI-native fabric for software delivery.

It brings together open-source tools for planning, building, operating, and improving modern software systems — especially SaaS, XaaS, and AI-assisted engineering workflows.

On GitHub, Cyber Fabric is organized around three main areas:

- **Cyber Constructor** — traceable software delivery from requirements to design, plans, and code.
- **Cyber Insight** — decision intelligence and engineering productivity analytics.
- **Cyber Ware** — reusable platform and application building blocks for modern SaaS/XaaS systems.

Each part can be used independently. Start with the repository that matches your current problem, then expand from there.

## Main repositories

### Cyber Constructor

Cyber Constructor helps teams keep requirements, design, plans, and code connected, reviewable, and traceable when working with AI coding tools.

- [`cyber-constructor`](https://github.com/cyberfabric/cyber-constructor) — main Cyber Constructor repository.
- [`cyber-constructor-app`](https://github.com/cyberfabric/cyber-constructor-app) — Cyber Constructor application repository.
- [`cyber-constructor-kit-sdlc`](https://github.com/cyberfabric/cyber-constructor-kit-sdlc) — SDLC kit for Cyber Constructor workflows, artifacts, and reviews.

Use this area if you want to structure AI-assisted development around explicit requirements, design artifacts, implementation plans, reviews, and validation.

### Cyber Insight

Cyber Insight collects operational data from engineering systems, resolves activity to unified identities, and helps teams understand productivity, bottlenecks, process performance, and team health.

- [`cyber-insight`](https://github.com/cyberfabric/cyber-insight) — main Cyber Insight backend and platform repository.
- [`cyber-insight-front`](https://github.com/cyberfabric/cyber-insight-front) — Cyber Insight frontend.

Use this area if you want visibility into how engineering work flows across tools, people, teams, and delivery processes.

### Cyber Ware

Cyber Ware provides reusable building blocks for building secure, modular, multi-tenant SaaS/XaaS systems.

- [`cyberware-rust`](https://github.com/cyberfabric/cyberware-rust) — Rust framework and middleware for enterprise-grade multi-tenant and multi-tier XaaS services.
- [`cyberware-frontx`](https://github.com/cyberfabric/cyberware-frontx) — UI development kit for modern SaaS applications.
- [`cyberware-csharp`](https://github.com/cyberfabric/cyberware-csharp) — C# modules and related Cyber Ware components.

Use this area if you are building platform services, SaaS backends, frontend components, or reusable application infrastructure.

## Supporting repositories

- [`DNA`](https://github.com/cyberfabric/DNA) — Development Norms & Architecture: principles, guidelines, and instructions for multi-tenant SaaS platform and application development.

## What should I do next?

Choose the path that matches your goal:

- **I want to move AI-assisted delivery beyond prototypes** — Start with [`cyber-constructor`](https://github.com/cyberfabric/cyber-constructor).
- **I want engineering productivity and delivery insights** — Start with [`cyber-insight`](https://github.com/cyberfabric/cyber-insight).
- **I want SaaS/XaaS building blocks instead of generating a whole app from scratch** — Start with [`cyberware-rust`](https://github.com/cyberfabric/cyberware-rust) and [`cyberware-frontx`](https://github.com/cyberfabric/cyberware-frontx).
