# Constructor Fabric

Constructor Fabric is an AI-native fabric for software delivery.

It brings together open-source tools for planning, building, operating, and improving modern software systems — especially SaaS, XaaS, and AI-assisted engineering workflows.

On GitHub, Constructor Fabric is organized around three main areas:

- **Studio** — traceable software delivery from requirements to design, plans, and code.
- **Insight** — decision intelligence and engineering productivity analytics.
- **Gears** — reusable platform and application building blocks for modern SaaS/XaaS systems.

Each part can be used independently. Start with the repository that matches your current problem, then expand from there.

## Main repositories

### Constructor Studio

Constructor Studio helps teams keep requirements, design, plans, and code connected, reviewable, and traceable when working with AI coding tools.

- [`studio`](https://github.com/constructorfabric/studio) — main Constructor Studio repository.
- [`studio-kit-sdlc`](https://github.com/constructorfabric/studio-kit-sdlc) — SDLC kit for Constructor Studio workflows, artifacts, and reviews.

Use this area if you want to structure AI-assisted development around explicit requirements, design artifacts, implementation plans, reviews, and validation.

### Constructor Insight

Constructor Insight collects operational data from engineering systems, resolves activity to unified identities, and helps teams understand productivity, bottlenecks, process performance, and team health.

- [`insight`](https://github.com/constructorfabric/insight) — main Constructor Insight backend and platform repository.
- [`insight-front`](https://github.com/constructorfabric/insight-front) — Constructor Insight frontend.

Use this area if you want visibility into how engineering work flows across tools, people, teams, and delivery processes.

### Constructor Gears

Constructor Gears provides reusable building blocks for building secure, modular, multi-tenant SaaS/XaaS systems.

- [`gears-rust`](https://github.com/constructorfabric/gears-rust) — Rust framework and middleware for enterprise-grade multi-tenant and multi-tier XaaS services.
- [`frontx`](https://github.com/constructorfabric/frontx) — UI development kit for modern SaaS applications.
- [`gears-csharp`](https://github.com/constructorfabric/gears-csharp) — C# modules and related Cyber Ware components.

Use this area if you are building platform services, SaaS backends, frontend components, or reusable application infrastructure.

## Supporting repositories

- [`DNA`](https://github.com/constructorfabric/DNA) — Development Norms & Architecture: principles, guidelines, and instructions for multi-tenant SaaS platform and application development.

## What should I do next?

Choose the path that matches your goal:

- **I want to move AI-assisted delivery beyond prototypes** — Start with [`studio`](https://github.com/constructorfabric/studio).
- **I want engineering productivity and delivery insights** — Start with [`insight`](https://github.com/constructorfabric/insight).
- **I want SaaS/XaaS building blocks instead of generating a whole app from scratch** — Start with [`gears-rust`](https://github.com/constructorfabric/gears-rust) and [`frontx`](https://github.com/constructorfabric/frontx).
